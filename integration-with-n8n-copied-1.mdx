### **O que é n8n?**

n8n é uma plataforma de automação de código aberto que permite criar fluxos de trabalho personalizados conectando diferentes serviços e APIs. Com sua flexibilidade, n8n é ideal para empresas que buscam soluções de automação personalizadas.

### **Benefícios da integração do Tess com n8n**

- Crie automações avançadas e personalizadas com Tess e outros serviços.
- Integre facilmente com APIs e sistemas internos.
- Reduza tarefas manuais e aumente a produtividade.
- Aproveite o poder do código aberto para adaptar integrações conforme necessário.

### **Passo a Passo: Acionando um Agente Tess AI do n8n**

Quer conectar n8n com Tess AI? Aqui está um guia rápido e amigável para você começar!

<Steps>
  <Step title="Crie um Novo Fluxo de Trabalho">
    Abra o n8n e inicie um novo fluxo de trabalho. Cada etapa (nó) representa uma ação ou serviço.
  </Step>
  <Step title="Adicione um Nó de Solicitação HTTP">
    Procure por "HTTP Request" no painel de nós e adicione ao seu fluxo de trabalho. Este nó enviará uma mensagem para seu agente Tess AI.
  </Step>
  <Step title="Configure a Solicitação HTTP">
    - **URL:** Use `https://api.tess.ai/agents/{agent_id}/execute` (substitua `{agent_id}` pelo ID do seu agente, encontrado na URL da plataforma Tess AI).
    - **Método:** Defina como `POST`.
    - **Cabeçalhos:** Adicione `Authorization: Bearer {your_token}` (gere seu token de API no Tess AI em "API Tokens").
    - **Corpo:** Defina como `JSON` e inclua a mensagem que deseja enviar, ex.:

      ```json
         {
         "temperature": "1",
         "model": "tess-5",
         "messages": [
             { "role": "user", "content": "Olá, como você pode me ajudar hoje?" }
         ],
         "tools": "no-tools",
         "wait_execution": true,
         "file_ids": [123, 321]
         }
      
      ```

      (Você pode usar expressões do n8n para inserir valores dinâmicos de nós anteriores, como mensagens do WhatsApp.)
  </Step>
  <Step title="Aguarde a Resposta do Agente">
    Certifique-se de incluir o parâmetro "wait_execution": true no corpo da solicitação. Isso diz ao Tess AI para aguardar a resposta do agente antes de continuar, para que você possa usar a resposta nas próximas etapas.
  </Step>
  <Step title="Teste Seu Fluxo">
    Salve e execute seu fluxo de trabalho. Verifique a saída do nó HTTP para ver a resposta do agente. Se algo estiver errado, verifique novamente seu token, ID do agente e formatação JSON.
  </Step>
  <Step title="Envie a Resposta de Volta">
    Quer responder a um usuário (ex.: no WhatsApp)? Adicione outro nó após a Solicitação HTTP para enviar a resposta do agente de volta.
  </Step>
</Steps>

<Tip>
  **Dicas:**

  - Mantenha seu token de API seguro—nunca o compartilhe publicamente!
  - Você pode remover parâmetros avançados da solicitação se eles já estiverem configurados em seu agente.
  - Sempre valide a entrada do usuário antes de enviá-la para o agente.
</Tip>

Para mais detalhes, consulte o [artigo de ajuda oficial do Tess AI](https://help.tessai.io/en/articles/135-%F0%9F%8E%A5como-criar-uma-requisicao-para-acionar-um-agente-na-tess-ai-usando-o-n8n%F0%9F%97%92%EF%B8%8F).
