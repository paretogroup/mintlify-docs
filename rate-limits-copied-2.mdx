---
title: "Límites de Tasa"
---

Los límites de tasa actúan como medidas de control para regular la frecuencia con la que los usuarios y aplicaciones pueden acceder a nuestra API dentro de períodos de tiempo especificados. Estos límites ayudan a garantizar la estabilidad del servicio, acceso justo y protección contra uso indebido.

### **Entendiendo los Límites de Tasa**

El límite de tasa predeterminado aplicado a todos los endpoints es **1 solicitud por segundo**.

Los límites de tasa se aplican a nivel de organización, no de usuarios individuales. Puedes alcanzar cualquier tipo de límite dependiendo de qué umbral alcances primero.

### **Manejo de Límites de Tasa**

Cuando excedes los límites de tasa, nuestra API devuelve un código de estado HTTP `429 Too Many Requests` con una respuesta como:

```
{
  "error": "Límite de tasa excedido",
  "retry_after": 60
}
```

### **Mejores Prácticas para Manejar Límites de Tasa**

1. **Implementar retroceso exponencial**: Al recibir una respuesta 429, espera el tiempo sugerido antes de reintentar.
2. **Almacenar respuestas en caché**: Almacena respuestas que no cambian frecuentemente para reducir llamadas a la API.
3. **Agrupar solicitudes**: Combina múltiples operaciones en una sola solicitud cuando sea posible.
4. **Monitorear uso**: Mantén un registro de tu uso de la API para evitar alcanzar límites inesperadamente.

## **Aumentos de Límite de Tasa**

Si necesitas límites de tasa más altos, contacta a nuestro equipo de soporte con:

- Tu caso de uso
- Volumen de solicitudes esperado
- Justificación para límites aumentados

Nuestro equipo revisará tu solicitud y trabajará contigo para encontrar una solución apropiada.
