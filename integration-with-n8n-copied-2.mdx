## **¿Qué es n8n?**

n8n es una plataforma de automatización de código abierto que permite crear flujos de trabajo personalizados conectando diferentes servicios y APIs. Con su flexibilidad, n8n es ideal para empresas que buscan soluciones de automatización personalizadas.

## **Beneficios de integrar Tess con n8n**

- Crea automatizaciones avanzadas y personalizadas con Tess y otros servicios.
- Integra fácilmente con APIs y sistemas internos.
- Reduce tareas manuales y aumenta la productividad.
- Aprovecha el poder del código abierto para adaptar integraciones según sea necesario.

## **Paso a Paso: Activando un Agente Tess AI desde n8n**

¿Quieres conectar n8n con Tess AI? ¡Aquí tienes una guía rápida y amigable para comenzar!

<Steps>
  <Step title="Crea un Nuevo Flujo de Trabajo">
    Abre n8n y inicia un nuevo flujo de trabajo. Cada paso (nodo) representa una acción o servicio.
  </Step>
  <Step title="Agrega un Nodo de Solicitud HTTP">
    Busca "HTTP Request" en el panel de nodos y agrégalo a tu flujo de trabajo. Este nodo enviará un mensaje a tu agente Tess AI.
  </Step>
  <Step title="Configura la Solicitud HTTP">
    - **URL:** Usa `https://api.tess.ai/agents/{agent_id}/execute` (reemplaza `{agent_id}` con el ID de tu agente, encontrado en la URL de la plataforma Tess AI).
    - **Método:** Configura como `POST`.
    - **Encabezados:** Agrega `Authorization: Bearer {your_token}` (genera tu token de API en Tess AI bajo "API Tokens").
    - **Cuerpo:** Configura como `JSON` e incluye el mensaje que quieres enviar, ej.:

      ```
         {
         "temperature": "1",
         "model": "tess-5",
         "messages": [
             { "role": "user", "content": "Hola, ¿cómo puedes ayudarme hoy?" }
         ],
         "tools": "no-tools",
         "wait_execution": true,
         "file_ids": [123, 321]
         }
      
      ```

      (Puedes usar expresiones de n8n para insertar valores dinámicos de nodos anteriores, como mensajes de WhatsApp.)
  </Step>
  <Step title="Espera la Respuesta del Agente">
    Asegúrate de incluir el parámetro "wait_execution": true en el cuerpo de la solicitud. Esto le dice a Tess AI que espere la respuesta del agente antes de continuar, para que puedas usar la respuesta en los siguientes pasos.
  </Step>
  <Step title="Prueba Tu Flujo">
    Guarda y ejecuta tu flujo de trabajo. Verifica la salida del nodo HTTP para ver la respuesta del agente. Si algo anda mal, verifica nuevamente tu token, ID del agente y formato JSON.
  </Step>
  <Step title="Envía la Respuesta de Vuelta">
    ¿Quieres responder a un usuario (ej.: en WhatsApp)? Agrega otro nodo después de la Solicitud HTTP para enviar la respuesta del agente de vuelta.
  </Step>
</Steps>

<Tip>
  **Consejos:**

  - ¡Mantén tu token de API seguro—nunca lo compartas públicamente!
  - Puedes eliminar parámetros avanzados de la solicitud si ya están configurados en tu agente.
  - Siempre valida la entrada del usuario antes de enviarla al agente.
</Tip>
