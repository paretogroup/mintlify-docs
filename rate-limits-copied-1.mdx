---
title: "Limites de Requisição"
---

Limites de Requisição são medidas de controle que regulam com que frequência usuários e aplicações podem acessar nossa API dentro de períodos de tempo determinados. Esses limites ajudam a garantir a estabilidade do serviço, o acesso justo e a proteção contra uso indevido.

### **Entendendo os Limites de Requisição**

O limite padrão aplicado a todos os endpoints é de **1 requisição por segundo**.

Os limites se aplicam no nível da organização, não de usuários individuais. Você pode atingir qualquer tipo de limite dependendo de qual threshold for alcançado primeiro.

### **Lidando com Limites de Requisição**

Quando você excede os limites, nossa API retorna o código de status HTTP `429 Too Many Requests` com uma resposta como:

```
{
  "error": "Rate limit exceeded",
  "retry_after": 60
}
```

### **Boas Práticas para Lidar com Limites de Requisição**

1. **Implemente exponential backoff**: Ao receber uma resposta 429, aguarde o tempo sugerido antes de tentar novamente.
2. **Faça cache das respostas**: Armazene respostas que não mudam com frequência para reduzir as chamadas à API.
3. **Agrupe requisições**: Combine múltiplas operações em uma única requisição sempre que possível.
4. **Monitore o uso**: Acompanhe o uso da sua API para evitar atingir os limites de forma inesperada.

### **Aumento de Limites de Requisição**

Se você precisar de limites mais altos, entre em contato com nosso time de suporte informando:

- Seu caso de uso
- Volume esperado de requisições
- Justificativa para o aumento dos limites

Nossa equipe analisará sua solicitação e trabalhará com você para encontrar a solução mais adequada.
