---
title: "Publish or unpublish execution page"
description: "Trigger publish or unpublish for a Tess Page by numeric page id, scoped to the workspace sent in x-workspace-id."
api: "POST https://api.tess.im/api/pages/{page_id}/publish"
authMethod: "bearer"
playground: "interactive"
---

Starts the publish or unpublish workflow for an existing **`AgentExecutionPage`** row. The page must belong to the workspace sent in **`x-workspace-id`**; the execution behind the page is resolved server-side to build storage paths and RPA payload.

Callers need **`workspace:pages:write`** or **`workspace:pages:manage`** (or the API-key fast path implemented in the controller). Otherwise the API returns **403** with a permission message.

### **Code examples**

<CodeGroup>

```http cURL
curl --request POST \
  --url 'https://api.tess.im/api/pages/120391/publish' \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --header 'x-workspace-id: YOUR_WORKSPACE_ID' \
  --data '{"unpublish":false}'
```

```json Node.js
const axios = require('axios');

axios
  .post(
    'https://api.tess.im/api/pages/120391/publish',
    { unpublish: false },
    {
      headers: {
        Authorization: 'Bearer YOUR_API_KEY',
        Accept: 'application/json',
        'Content-Type': 'application/json',
        'x-workspace-id': 'YOUR_WORKSPACE_ID'
      }
    }
  )
  .then((response) => response.data);
```

```python Python
import requests

url = "https://api.tess.im/api/pages/120391/publish"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Accept": "application/json",
    "Content-Type": "application/json",
    "x-workspace-id": "YOUR_WORKSPACE_ID",
}

response = requests.post(url, json={"unpublish": False}, headers=headers)
response.status_code, response.json()
```

</CodeGroup>

### **Path parameters**

<ParamField path="page_id" type="integer" required>
  Database id of the **`agent_execution_pages`** row. Unknown ids return **404** (`Page not found.`).
</ParamField>

### **Headers**

<ParamField header="x-workspace-id" type="integer" required>
  Must match the workspace that owns the execution behind the page. Wrong workspace → **403** (`You do not have access to this workspace or it does not exist`). Missing header → **400**.
</ParamField>

### **Body**

<ParamField body="unpublish" type="boolean">
  Set to **`true`** to unpublish; omit or **`false`** to publish. Unpublishing clears the generated file URL before enqueueing RPA.
</ParamField>

### **Successful response**

HTTP **200** with the refreshed page snapshot:

```json
{
  "id": 120391,
  "subdomain": "my-demo-page",
  "status": "active",
  "url": "https://my-demo-page.pages.tess.im",
  "updated_at": "2026-04-18T12:40:00.000000Z"
}
```

After **`unpublish: true`**, **`status`** becomes **`inactive`** when RPA completes (same response shape; values reflect post-refresh state).

### **Common errors**

| Status | Meaning |
|--------|---------|
| **400** | Missing **`x-workspace-id`**. |
| **403** | Workspace mismatch or missing **write/manage** permission. |
| **404** | Unknown **`page_id`**, missing execution, or **no `index.html`** in the resolved workspace storage path when publishing. |
| **500** | RPA/workflow dispatch failed; retry later. |

### **Panel parity**

Interactive users still call **`POST /backend-api/executions/{execution_id_or_slug}/pages/publish`** from the UI; that route now answers with **200** and the same style of JSON payload instead of **202** `queued`.
