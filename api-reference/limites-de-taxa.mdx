---
title: "Limites de Requisição"
---

Os limites de requisição atuam como medidas de controle para regular a frequência com que usuários e aplicações podem acessar nossa API dentro de períodos de tempo especificados. Esses limites ajudam a garantir estabilidade do serviço, acesso justo e proteção contra uso indevido.

## **Entendendo os Limites de Requisição**

O limite de taxa padrão aplicado a todos os endpoints é **1 requisição por segundo**.

Os limites de taxa se aplicam no nível da organização, não de usuários individuais. Você pode atingir qualquer tipo de limite dependendo de qual limite você alcançar primeiro.

## **Lidando com Limites de Taxa**

Quando você excede os limites de taxa, nossa API retorna um código de status HTTP `429 Too Many Requests` com uma resposta como:

```
{
  "error": "Limite de taxa excedido",
  "retry_after": 60
}
```

### **Melhores Práticas para Lidar com Limites de Taxa**

1. **Implementar backoff exponencial**: Ao receber uma resposta 429, aguarde o tempo sugerido antes de tentar novamente.
2. **Armazenar respostas em cache**: Armazene respostas que não mudam frequentemente para reduzir chamadas à API.
3. **Agrupar requisições**: Combine múltiplas operações em uma única requisição quando possível.
4. **Monitorar uso**: Acompanhe seu uso da API para evitar atingir limites inesperadamente.

## **Aumentos de Limite de Taxa**

Se você precisar de limites de taxa mais altos, entre em contato com nossa equipe de suporte com:

- Seu caso de uso
- Volume de requisições esperado
- Justificativa para limites aumentados

Nossa equipe revisará sua solicitação e trabalhará com você para encontrar uma solução apropriada.
